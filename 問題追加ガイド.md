# 問題追加ガイド

AlgoBite に問題を追加するときの手順書。
問題は2種類あり、追加場所と必須項目が違う。

- **穴埋め（Puzzle）** … コードの `{{...}}` を選択肢から埋める形式
- **並べ替え（Reorder）** … タイルを正しい順にタップする形式

両方とも、追加すれば **「今日のひと口」のローテーション** と **復習モードの一覧** に自動で組み込まれる（手動登録は不要）。出題は `穴埋め3 : 並べ替え1` の比率でインターリーブされる（`GameViewModel.dailyChallenges`）。

---

## 共通ルール（必ず守る）

| 項目 | ルール |
|---|---|
| `id` | **全問でユニーク**なケバブケース文字列（例 `two-sum`）。アニメや絵文字のマッピングキーになるので後から変えない。 |
| `difficulty` | **`"Easy"` / `"Medium"` / `"Hard"` のいずれか**（この3文字列のみ。色分岐がこの値を見ている）。 |
| `topic` | 日本語/英語どちらでも可。`" / "` 区切りで複数可（例 `"Two Pointers / String"`）。**先頭の語**が統計のトピック集計・イラスト・アニメ判定に使われる。 |
| `explanation` | **必須**（穴埋め・並べ替えとも省略不可。書かないとビルドエラー）。**必ず「考え方」を書く。問題文の言い換えはNG**（詳細は下の「ポイント欄の書き方」）。 |
| 絵文字 | UI本文には絵文字・💡などの装飾アイコンを使わない方針（テキストに混ぜない）。 |

## 1. 穴埋め（Puzzle）の追加

### 追加場所
`AlgoBite/Problems.swift`

```swift
struct PuzzleData {
    static let all: [PuzzleProblem] = p1_50 + p51_100   // ← この2配列の中身が全問
}
```

`p1_50` か `p51_100` のどちらかの配列の `[ ... ]` の中に、`PuzzleProblem(...)` を**カンマ区切りで**追記する（配列の末尾要素にはカンマ不要なので注意）。

### テンプレート（コピーして使う）

```swift
    // NN. タイトル
    PuzzleProblem(
        id: "my-problem-id",                       // ユニーク・ケバブケース
        title: "My Problem",
        difficulty: "Easy",                        // Easy / Medium / Hard
        topic: "Hash Map",                         // 先頭語が分類に使われる
        prompt: "何をする問題かを1〜2文で。",
        example: "例: input  →  output",
        template: [
            "def solve(nums):",
            "    seen = {}",
            "    for i, n in enumerate(nums):",
            "        x = {{diff}}",                 // 穴は {{キー}} で書く
            "        seen[n] = {{val}}"
        ],
        slots: [
            // template の {{キー}} と完全一致するキーで定義する
            "diff": PuzzleSlot(id: "diff", label: "差分",   answer: "target - n", choices: ["target - n", "n", "target"]),
            "val":  PuzzleSlot(id: "val",  label: "辞書値", answer: "i",          choices: ["i", "n", "0"])
        ],
        explanation: "各要素について target との差を辞書で探す。差が既出ならペア成立、無ければ自分を登録。1パス・ハッシュで O(n)。"
    ),
```

### 穴埋めの必須チェック（ここを間違えると壊れる）

1. **`{{キー}}` と `slots` のキーは1対1で一致**させる。
   - template に `{{diff}}` があるのに slots に `"diff"` が無い → スロットが空表示で詰む。
   - slots に余分なキーがあっても出題には出ない（無害だが消す）。
2. **`answer` は必ず `choices` の中に含める**。含まれないと正解にたどり着けない。
3. `choices` は **3個**を推奨（既存と統一）。順序はシャッフルされないので、正解が先頭固定にならないよう散らす。
4. **穴の出題順は template に登場する順**（`orderedSlotIDs` が `{{...}}` を上から拾う）。解説アニメもこの順で再生される。
5. `template` の各行は1行＝1コード行の文字列。インデントは空白で表現する。

### アニメーション（任意だが推奨）
解説の「動きで見る」アニメは `AlgoBite/TopicAnimations.swift` の `topicAnimation(for:)` で **`problem.id` 単位**にマッピングされている。

- **何も足さなくても動く**：id にマッチが無ければ `topic` のキーワードから汎用アニメに自動フォールバックする。
- 専用アニメを付けたい場合のみ、`topicAnimation(for:)` の `switch` に
  ```swift
  case "my-problem-id": MyCustomAnim()
  ```
  を追加し、対応する View を実装する。

> ホームカードの抽象イラスト（`TopicIllustration`）は `topic` 文字列から自動判定なので、追加作業は不要。

## 2. 並べ替え（Reorder）の追加

### 追加場所
- 定義：`AlgoBite/Extras/ReorderQuizData.swift`（個別の `static let`）
- 登録：同ファイルの `ReorderQuiz.allList` 配列に**名前を追記**（ここに入れないと出題されない）

### テンプレート（コピーして使う）

```swift
    static let myReorderQuiz: ReorderQuiz = .init(
        id: "my-reorder-id",                       // ユニーク・ケバブケース
        title: "並べ替えのタイトル",
        topic: "ソート",                            // 分類に使われる
        prompt: "どういう並びにするかの指示文。",
        pool: ["1", "2", "3", "4", "5"],           // タップ候補（表示される全タイル）
        answer: ["2", "4", "1", "3", "5"],         // 正解の並び順
        explanation: "なぜその並びになるのかの考え方。"
    )
```

そして `allList` に追記：

```swift
    static let allList: [ReorderQuiz] = [
        .bubbleSortPass,
        ...
        .myReorderQuiz,      // ← 追加
    ]
```

### 並べ替えの必須チェック

1. `explanation` は**必須フィールド**（省略するとビルドエラー。穴埋めも同様に必須）。
2. **`answer` の各要素は `pool` に存在する**こと（同じ値の重複もOK。重複は LCS 判定で正しく扱われる）。
3. `pool` と `answer` は**要素数が同じ**になるのが基本（全タイルを使い切る前提）。
4. 採点は「正解との最長共通部分列(LCS)」で行う。**順序が合っている要素は緑、ずれている要素だけ赤**になって戻ってくる、という UI なので、答えは「一意に決まる並び」にする。
5. 絵文字（一覧の見出しに出る）は `ReorderQuizData.swift` の `var emoji` の `switch id` に追加できる。**未指定なら `📋` がデフォルト**で出る（必須ではない）。

> ⚠️ `bubbleSortPass` だけは歴史的経緯で `ReorderQuiz.swift` 側に定義されている。新規は **`ReorderQuizData.swift` に統一**して追加すること。

## 3. 追加後の確認手順

### ビルド確認
```bash
xcodebuild -scheme AlgoBite \
  -destination 'platform=iOS Simulator,name=iPhone 17' \
  -configuration Debug build 2>&1 | grep -E "error:|BUILD SUCCEEDED|BUILD FAILED"
```

### 件数・解説の網羅チェック
```bash
cd AlgoBite

# 穴埋めの総数と、解説が入っている数（必ず一致させる）
grep -c "PuzzleProblem(" Problems.swift
grep -c "explanation:" Problems.swift

# 並べ替えの総数と解説数
grep -c "\.init(" Extras/ReorderQuizData.swift     # 定義数
# allList に入れ忘れていないかは目視 or 下記で
grep -c "explanation:" Extras/ReorderQuizData.swift
```
- 穴埋め：`PuzzleProblem(` の数 == `explanation:` の数 になっていればOK（全問に解説あり）。
- 並べ替え：定義した数すべてが `allList` に入っているか確認。

### 動作確認（任意）
DEBUG ビルドは `DebugCapture.pinnedProblemID`（現在 `climbing-stairs`）に出題が固定される。
特定の問題を画面で確認したいときは、`AlgoBite/DebugCapture.swift` の `pinnedProblemID` を一時的にその id へ変更して実行する（**確認後は必ず戻す**）。

## 「ポイント」欄（explanation）の書き方

`explanation` は問題画面の **「ポイント」カード**に表示される。
**問題文の言い換えは禁止**。次の3つを2〜3文で。

1. **核心のアイデア**（なぜこの手法か／何を使うか）
2. **要点となる一手**（穴埋めの答えがなぜそうなるか、ループ条件の意味など）
3. **計算量**（例 `O(n)`、`O(n log n)`、空間 `O(1)` など）

| ❌ ダメな例（問題文の言い換え） | ✅ 良い例（考え方） |
|---|---|
| 「n段の階段を1または2段ずつ登る方法の数を返す」 | 「dp[i]=dp[i-1]+dp[i-2]。i段目には『1段前から1歩』か『2段前から2歩』で来るので、フィボナッチと同じ漸化式になる。O(n)。」 |
| 「重複しない最長部分文字列の長さを返す」 | 「右端を伸ばし、重複文字が入ったら左端を縮めて集合から除く。ウィンドウ幅 right-left+1 の最大を記録する。O(n)。」 |

## まとめ：最短手順

**穴埋めを足す**
1. `Problems.swift` の `p1_50` か `p51_100` に `PuzzleProblem(...)` を追記
2. `{{キー}}` ⇔ `slots` キー一致、`answer` ∈ `choices` を確認
3. `explanation` に考え方を記入
4. ビルド＋件数チェック

**並べ替えを足す**
1. `ReorderQuizData.swift` に `static let xxx: ReorderQuiz = .init(...)` を定義
2. **`allList` に `.xxx` を追記**（忘れがち）
3. `answer` の要素が `pool` にあるか確認、`explanation` 記入
4. ビルド＋件数チェック
